Trend 01  ·  Reasoning / Test-Time Compute
04 · 16
Trend 01  ·  考える時間を買う

推論は、
時間をかけるほど
高度に。

学習時のスケーリングに加え、推論時スケーリング（Test-Time Compute）が主流に。
同じモデルでも「考える時間」を伸ばすほど数学・コード・科学の性能が上がる、という実証が2025年に固まった。
2026年はExtended Thinkingが全モデルで標準装備。コストは「1回の応答」ではなく「1回の思考時間」で積まれる。

モデル選びは「パラメータ数」から
「思考予算（thinking budget）」へ。

Trend 02  ·  開発は「書く」から「頼む」へ

コードは書かない。
"書かせる"。

Trend 03  ·  Protocol Layer
06 · 16
Protocols · 接続の標準化

AIは、プロトコルで繋がる。

2025年に仕様が固まったMCPとA2Aは、2026年になってエコシステムの前提になった。
モデルはMCPでツールを握り、エージェントはA2Aで会話する。この2層分離が整理されたことで、スタックは一気にモジュール化された。

MCP · Model Context Protocol
AIにツールを
渡すための標準。
Anthropic発、AI版USB-Cとも呼ばれる接続規格。ファイル・DB・API・業務ツールを、モデルから均一に叩ける。
2026年はOS・IDE・SaaSへの組込みが当たり前になり、個人の開発環境にも浸透。
発祥：2024年末 レイヤー：Model ↔ Tool
A2A · Agent-to-Agent
AI同士の
会話の標準。
Google主導、50社超が参加して2025年に仕様公開。
Agent Card / Task / Artifact という共通スキーマで、別ベンダー製エージェントが協調可能に。
MCPが「ツールを持たせる」のに対し、A2Aは「別の頭脳と連携させる」層。
発祥：2025年4月 レイヤー：Agent ↔ Agent
